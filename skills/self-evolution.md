# Self-Evolution — 自进化框架

## 适用场景
想让 AI Agent 能自主发现能力短板→学习弥补→固化经验。适合追求 Agent 持续自优化的高级用户。

## 核心理念

AI 不应等待人工调试，而是能自主走完：发现问题 → 定位根因 → 提出方案 → 修改 → 验证 → 固化的完整闭环。

## 8 阶段循环

### Phase 1 — Review（回顾）
扫描近期的操作日志和 Trace，提取：
- 错误模式（什么操作反复失败）
- 用户纠正（用户反复提醒什么）
- 能力瓶颈（什么任务做不好）
- 重复劳动（什么操作可自动化）

### Phase 2 — Ideate（构思）
基于证据提出改进方案。每个方案回答：
```
问题: 观察到什么？
根因: 为什么发生？
方案: 改什么能让它不再发生？
验证: 怎么确认改对了？
```

### Phase 3 — Modify（修改）
- 一轮只改 1 件事
- ≤5 个文件
- 改前先备份

### Phase 4 — Commit（提交）
先 Git commit 再验证。提交信息写清楚：
```
修改: 文件名/位置
依据: 数据/Trace/证据
目的: 解决什么问题
```

### Phase 5 — Verify（验证）
- L1：语法检查 + 结构完整性
- L2：功能验证（curl/运行测试）
- L3：回归检查（改了 A，B/C 正常吗）

### Phase 6 — Gate（门控）
5 维检查，全部通过才保留：
- ✅ 安全（没有引入新漏洞）
- ✅ 稳定（没有破坏现有功能）
- ✅ 效果（确实解决了问题）
- ✅ 性能（没有明显变慢）
- ✅ 规范（代码风格一致）

任一不通过 → 自动回滚。

### Phase 7 — Log（记录）
把这次进化的经验写入知识库：
```
- 修了什么
- 根因是什么
- 验证方式
- 下次怎么预防
```

### Phase 8 — Loop（循环）
回到 Phase 1，继续发现下一个改进点。

## 原则

1. **AI 做单点原子修改**，不批量改
2. **Git 控制流程**，改坏了就回滚
3. **基于 Trace 诊断**，不靠猜
4. **验证 > 自信**，通过测试才算数
