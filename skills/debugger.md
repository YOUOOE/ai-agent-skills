# Debugger — 系统化调试

## 适用场景
遇到 Bug 时，不要猜，按流程来。

## 四阶段

### Phase 1 — 复现
- 精确记录复现步骤
- 确认环境一致性（本地 vs 生产）
- 确认触发条件（什么输入、什么状态才会出）

### Phase 2 — 圈定范围

| 方法 | 做法 | 适合 |
|------|------|------|
| 二分法 | 从中间步骤断点，确认前半还是后半出问题 | 多步操作/MVP |
| 最小化 | 写最小可复现代码，排除干扰因素 | 复杂系统/库调用 |
| 对照法 | diff 对比上一个能用的版本 | 回归/升级/迁移 |
| 数据追踪 | 输入→每一步中间状态→输出，逐段验证 | 数据处理/API 调用 |

### Phase 3 — 推演根因
列出 3-5 种可能根因，按概率排序，逐个排除：

```
可能性 A（60%） — xxx
  验证方式: curl xxx 看返回
可能性 B（25%） — yyy
  验证方式: 加日志看变量值
可能性 C（10%） — zzz
  验证方式: 回退版本测试
```

从最可能路径开始验证，不跳过。

### Phase 4 — 最小修复
- 只改根因，不顺手改无关代码
- 改完立即验证修复有效
- 检查同模式是否在别处也存在

## 输出模板

```
## 现象
{描述}

## 根因
{定位结果}

## 修复
{改了什么 + 验证方式}

## 预防
{怎么避免再犯}
```

## 陷阱
- ❌ 不看日志直接猜根因
- ❌ 修了表象就停，不追根因
- ❌ 改完不验证关联功能
- ❌ 复现不了就放弃

## 示例

```
现象: 用户登录后显示白页，F12 无报错

二分定位:
  - 登录 API 返回 200 ✓
  - 首页 HTML 渲染正常 ✓
  - JS 执行到 100 行后停止 →
  - 发现 iframe 跨域 sandbox 阻止了 localStorage

根因: iframe sandbox 属性缺少 allow-same-origin
修复: 添加 allow-same-origin 到 sandbox 属性

验证: 重新登录，页面正常渲染
预防: 检查所有 iframe 的 sandbox 属性完整性
```
