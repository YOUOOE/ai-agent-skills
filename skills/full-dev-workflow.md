# Full Dev Workflow — 完整开发工作流

## 适用场景
接到一个新功能开发任务。按流程走，减少返工、提高质量。

## 6 阶段流水线

### 1. Explore — 探索
```
读项目 README → 扫目录结构 → 找入口文件
读配置文件（package.json/pyproject.toml）
理解数据流（请求→处理→存储→返回）
```

### 2. 实现 — 编码
```
改前备份 → 改代码 → 语法检查
   ↓
功能验证（curl/api 测试）
   ↓
关联检查（改了 A，B/C/D 还能用吗？）
```

### 3. 审查 — Code Review
```
逐文件读 diff → 按 checklist 过
P0: 安全/资金/数据 — 必须修
P1: 逻辑/性能/兼容 — 建议修
P2: 命名/注释/风格 — 可选
```

### 4. 测试 — 补测试
```
核心路径有没有测试？
Bug 修复有没有补回归测试？
边界条件覆盖了吗？
```

### 5. 简化 — 整理代码
```
有重复代码吗？
函数职责单一吗？
命名清晰吗？
```

### 6. 安全 — 上线前
```
有硬编码密钥吗？
越权漏洞？
敏感信息泄漏？
```

## 输出

每个阶段完成后输出一句话结论：

```
探索完成: 项目是 FastAPI + Vue3 单体应用，核心入口 main.py
实现完成: 新增 /api/orders/history 端点，curl 200 ✓
审查完成: 1 个 P0（SQL注入风险），已修
测试完成: 核心路径 3 条测试，全部通过
简化完成: 抽取 1 个公共函数，删除 2 处重复代码
安全完成: 无遗留安全问题
```
