# Vibe Coding 方法论

用自然语言驱动AI编程的完整工作流。适合快速原型开发、工具类页面、创意项目。

## 核心理念

"fully give in to the vibes" — 用自然语言描述，让AI生成代码，人类只做审核和方向调整。

## 工作流

```
说需求 → AI生成 → 看结果 → 调整 → 再生成 → 满意 → 上线
```

### 有效描述技巧
- **给具体场景**：不说"做个登录页"，说"一个深色背景的登录页，有邮箱/密码输入框和验证码，手机适配"
- **给参考方向**：说"像Stripe那样的简洁风格"比说"好看"有效
- **一次一件事**：每次提一个明确的改动方向
- **给对比反馈**：说"左边间距太大了，缩一半"比说"调一下"精确

### 约束管理
在项目根目录放 `AGENTS.md` 或 `.cursorrules`：
```markdown
# 项目规范
- 使用Python 3.10+，所有函数加类型注解
- 前端用Tailwind CSS，响应式优先
- 禁止硬编码API Key
- 修改前先备份
```

### 验证循环
```
AI写完 → 立即运行 → 看到效果 → 评估 → 继续调整
```

## 适用范围

| 适合 | 不适合 |
|------|--------|
| 前端页面/UI原型 | 核心算法实现 |
| 工具类脚本 | 安全敏感代码 |
| API集成 | 支付/认证逻辑 |
| 数据可视化 | 数据库迁移方案 |
| 原型验证 | 性能关键路径 |

## 最佳实践
1. **小步快跑**：每次只让AI做一个小改动，验证后再继续
2. **运行验证**：生成代码后必须实际运行确认
3. **清晰描述**：像给设计师描述需求一样给AI描述
4. **建立约束**：用AGENTS.md定义风格和规范
