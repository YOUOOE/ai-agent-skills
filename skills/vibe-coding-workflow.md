# Vibe Coding Workflow

## 概述

Vibe Coding 是由 Andrej Karpathy 提出的 AI 编程新范式——人类出题+审核，AI 负责解题+改卷。你只说想要什么，不需要关心怎么实现。

## 核心理念

```
传统编程: 需求 → 设计 → 编码 → 测试 → 部署
                     ↑
               全靠手写代码

Vibe Coding: 需求 → AI 生成 → 你审核 → 再提要求 → AI 再改
                     ↑
               全靠自然语言描述
```

## 工作流

### 1. 清晰描述需求

```markdown
## 好的描述示例

❌ "帮我做个页面"
→ 太模糊，AI 不知道做成什么样

✅ "做一个 AI 图片生成工具的首页，深色工业风，
   顶部导航栏（Logo + 4个菜单），英雄区是粒子背景+大字标题，
   中间展示 3 张示例生成图片，底部三栏功能介绍。
   手机端自适应，所有图标用 SVG 线形图标。"
→ AI 能精确理解你想要什么

关键要素：
- 风格：深色/浅色/工业风/简洁
- 结构：顶部有什么，中间有什么，底部有什么
- 功能：按钮点哪里，跳转到哪里
- 适配：电脑/手机/平板
- 约束：不要什么、用什么技术栈
```

### 2. 迭代式审核

```markdown
## 审核要点

第一轮：看整体感觉对不对
- 风格符合预期吗？
- 布局结构对吗？
- 有没有明显的 bug？

第二轮：看细节
- 按钮 hover 有效果吗？
- 手机端排版正常吗？
- 主题切换工作吗？

第三轮：看边缘情况
- 加载中显示 skeleton 吗？
- 空数据有提示吗？
- 出错时有友好错误吗？

说需求 → 看结果 → 调整 → 再出结果
每次只调一个方向，别一次性提太多要求
```

### 3. 小步快跑原则

```python
# 错误方式：一次性说太多
"帮我做一个完整的电商平台，包含用户系统、
 商品管理、购物车、支付、订单、后台..."

# 正确方式：分步迭代
Step 1: "做一个商品列表页，从 JSON 数据渲染卡片"
Step 2: "给卡片加点击查看详情弹窗"
Step 3: "加购物车功能，用 localStorage 存"
Step 4: "加结算页面"
```

### 4. 常用约束模板

```markdown
## 技术约束

技术栈：
- 纯 HTML/CSS/JS，不要框架
- 所有图片本地路径，不用 CDN
- 图标用内联 SVG 或 Font Awesome

设计约束：
- 深色主题（背景 #0d0d12，文字 #e8e8ed）
- 主色青色 #01c2c3
- 圆角 8-12px
- 手机端微信/QQ 聊天气泡风格

功能约束：
- 所有数据从 API 获取
- 操作有 loading 状态
- 错误有友好提示
- 手机端和桌面端体验一致
```

### 5. 验证清单模板

```markdown
- [ ] 页面打开无报错（控制台检查）
- [ ] 所有链接可点击（无 404）
- [ ] 手机端排版正常（@media 响应式）
- [ ] 主题切换不闪烁
- [ ] 加载/空/错误三态覆盖
- [ ] 所有按钮有 hover/active 效果
```

## 常见陷阱

### ⚠️ 需求说得不够细
- "做个登录页" → AI 可能做成亮的，你要暗的
- 修复：把你想要的样子描述到别人能画出来的程度

### ⚠️ 一次性提太多修改
- "把 A 改大，B 颜色换掉，C 加个功能，D 要圆角"
- 修复：一次只改 1-2 件事，确认后再改剩下的

### ⚠️ 不验证就上线
- AI 生成的代码可能有语法错误
- 修复：上线前必须全链路过一遍

## 适用场景

| 场景 | 适合 | 不适合 |
|------|------|--------|
| 原型/演示 | ✅ 快速出效果 | ❌ 性能要求高 |
| 内部工具 | ✅ 功能优先 | ❌ 安全性极高 |
| 营销页面 | ✅ 视觉优先 | ❌ 复杂交互 |
| 新功能探索 | ✅ 试方向 | ❌ 支付/资金 |
