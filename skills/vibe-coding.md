# Vibe Coding — AI 协程编程

## 适用场景
快速原型、工具类页面、创意项目。适合"说需求让 AI 做"的工作模式。

## 核心理念

> "Fully give in to the vibes" — Andrej Karpathy

用自然语言描述需求，AI 生成代码，人类做审核和方向调整。不纠结每一行代码怎么写。

## 工作流

```
说需求 → AI 生成 → 跑起来看效果
                 ↓
           满意？→ 是→ 上线
                 ↓ 否
            调整需求 → 继续
```

## 有效描述的技巧

| ❌ 模糊描述 | ✅ 精确描述 |
|-------------|-------------|
| "做一个登录页" | "一个深色背景的登录页，邮箱/密码输入框 + 验证码，手机适配" |
| "改好看一点" | "左边距缩小一半，字体加粗到 600，背景换深灰色" |
| "加个动画" | "页面加载时，标题从上方 fadeIn + slideDown，1.2 秒缓出" |

## 约束管理

项目根目录放 `.cursorrules` 或 `AGENTS.md`：

```markdown
# 项目规范
- Python 3.10+，所有函数加类型注解
- 前端用 Tailwind CSS，响应式优先
- 禁止硬编码 API Key
- 修改前先备份
```

## 适用范围

| 适合 | 不适合 |
|------|--------|
| 前端页面/UI 原型 | 核心算法/安全敏感代码 |
| 工具类脚本/API 集成 | 支付/认证逻辑 |
| 数据可视化 | 数据库迁移方案 |
| 快速原型验证 | 需要精确性能控制的地方 |

## 原则

1. **小步快跑** — 每次只让 AI 做一个小改动，验证后再继续
2. **运行验证** — 生成代码后必须实际跑起来确认效果
3. **清晰描述** — 像给设计师描述需求一样描述
4. **建立约束** — 用 AGENTS.md 定义风格和规范
