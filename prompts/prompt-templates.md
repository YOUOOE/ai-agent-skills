## 代码助手

### 代码审查
```
你是一个严格的代码审查者。审查以下代码：

{代码}

请检查：
1. 逻辑正确性（边界条件、异常处理）
2. 安全性（注入、硬编码、权限）
3. 可维护性（命名、职责单一、重复）
4. 性能（N+1查询、循环效率）

按 P0(必须修) / P1(建议修) / P2(可选的) 分级输出。
```

### 调试助手
```
我遇到了一个Bug。以下是上下文：

**现象**: {现象}
**代码**: 
```{语言}
{代码}
```
**错误信息**: {错误信息}
**已尝试**: {已尝试的方案}

请帮我：
1. 分析可能的根因（列出3-5种可能性，按概率排序）
2. 给出验证每种的方案
3. 确定根因后给出修复方案
```

### 代码转文档
```
为以下代码生成文档。包含：
1. 函数/类的作用（一句话）
2. 参数说明（类型、含义、默认值）
3. 返回值说明
4. 使用示例

代码：
{代码}
```

## 写作助手

### 文章润色
```
请润色以下文章，保持原意但提升表达质量：

{文章}

要求：
1. 保持原文风格和专业度
2. 优化句式结构，避免长句堆砌
3. 修正用词不当
4. 检查逻辑连贯性
5. 输出时标记修改了哪些地方
```

### 内容改写
```
将以下内容改写成{目标平台}风格：

原始内容：{内容}

{目标平台}特点：
- 小红书：emoji分段、亲测语气、标签结尾
- 知乎：专业客观、引经据典、段落清晰
- 公众号：标题党开头、故事化叙述、金句收尾
- 推特：极简有力、140字内核心观点
```

## 设计助手

### UI评审
```
作为UI/UX专家，评审以下设计：

设计描述/截图描述：{设计}

从以下维度评价：
1. 视觉层次 — 信息优先级是否清晰？
2. 一致性 — 配色/字体/间距统一吗？
3. 可用性 — 用户操作路径短吗？
4. 可访问性 — 对比度/字号达标吗？
5. 响应式 — 手机端能正常使用吗？

给出1-5分的评分和改进建议。
```

### 设计规范生成
```
为一款{产品类型}产品生建设计规范。

品牌调性：{品牌调性}
目标用户：{目标用户}

请输出：
1. 配色方案（主色/辅助色/中性色/功能色，带色值和用途）
2. 字体体系（标题/正文/标注的字号字重行高）
3. 间距规范（栅格系统/间距单位）
4. 圆角规范
5. 组件风格（按钮/输入框/卡片/弹窗）
```

## 分析助手

### 竞品分析
```
分析以下产品的竞争力：

目标产品：{产品名}（{一句话描述}）
竞品：{竞品列表}

分析维度：
1. 核心功能对比 — 哪些功能有/无，质量如何
2. 用户体验 — 上手难度、操作效率、学习成本
3. 商业模式 — 收费方式、定价策略
4. 技术路线 — 技术栈、架构方案
5. 差异化 — 我们做什么他们不做，他们做什么我们不做

输出为对比表格。
```

### 根因分析（5Why）
```
用5Why法分析以下问题：

问题：{问题}

现象描述：{现象描述}

请逐层追问Why，直到找到系统级根因。
每层给出证据（数据/代码/日志引用）。
最后给出可落地的修复方案。
```

## 学习助手

### 概念解释
```
用费曼技巧解释以下概念：

概念：{概念}
目标受众：{受众水平（小学生/初中生/高中生/大学生/从业者）}

请：
1. 用最简单的语言一句话说明
2. 拆解核心组成部分
3. 用类比/例子帮助理解
4. 指出常见的误解
5. 给出进一步学习的方向
```

### 知识总结
```
将以下内容总结为结构化的知识卡片：

{内容}

输出格式：
- 核心观点（1-3句话）
- 关键细节（5-8个要点）
- 实践建议（3-5条可操作建议）
- 延伸阅读（推荐关键词/话题）
```

## Agent系统提示词

### 通用助手身份
```
你是{助手名}，一个{角色定义}。

能力范围：
- {能力1}：{简要说明}
- {能力2}：{简要说明}
- {能力3}：{简要说明}

行为准则：
1. 简洁直接，结论先行
2. 不知道的事直接说不知道
3. 关键信息可以自然带来源
4. 专注于解决问题，不闲聊

记住：用户的时间比你的完整度更重要。
```

### 代码助手身份
```
你是{助手名}，一个代码助手。

核心原则：
1. **改了不验证=没改完** — 改完代码后必须验证语法/运行
2. **一次做精不返工** — 想清楚了再动手
3. **诚实透明** — 做不到直接说，不画饼

回复格式：
- 先给结论（1-3句话）
- 需要时给代码/方案
- 复杂问题附上验证步骤
```
