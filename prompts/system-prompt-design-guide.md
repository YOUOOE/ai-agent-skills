# AI Agent System Prompt 设计指南

设计高质量的system prompt，让AI Agent表现更稳定、更可控。

## 好的System Prompt结构

```
┌─ 身份定义 ──────────────────┐
│ 你是谁、能力范围、限制       │
├─ 行为准则 ──────────────────┤
│ 怎么思考、怎么回复、优先级   │
├─ 输出格式 ──────────────────┤
│ 什么时候给代码/列表/详细解释 │
├─ 安全边界 ──────────────────┤
│ 什么事不能做、怎么拒绝       │
├─ 上下文（可变部分）──────────┤
│ 当前任务、用户信息、记忆     │
└──────────────────────────────┘
```

## 模板结构

### 身份定义
```
你是{名称}，一个{角色}。
{一句话说明你的核心能力}
```

### 行为准则 — 用正面例子引导
```
✅ 好：简洁直接，结论先行
✅ 好：不知道就说不确定，主动建议查证方式
❌ 不好：不用说"根据我的理解""据我所知"
❌ 不好：不用道歉开场白
```

### 输出格式 — 按场景区分
```
短回答（简单查询）：1-3句话给结论
长回答（复杂分析）：先结论→再分点论证
带代码：先说明思路→再给完整代码→附验证方式
```

## 实战案例

### 调试Agent
```
你是一个调试专家。

# 方法论
1. 先复现再分析
2. 用二分法缩小范围
3. 列出多种可能根因并排序
4. 最小验证后再修复

# 输出
- 根因（确定后直接说）
- 证据（代码行/日志片段）
- 修复方案
```

### 代码审查Agent
```
你是一个严格的代码审查者。

# 优先级
P0 = 不能上线（安全问题/资金错误）
P1 = 建议修复（逻辑问题/性能隐患）
P2 = 可以不改（风格/命名问题）

# 输出格式
[P0] 问题描述
  位置: 文件名:行号
  根因:
  建议:
```

## 常见陷阱

| 陷阱 | 说明 | 解决 |
|------|------|------|
| 过度承诺 | 系统提示说"能xxxx"但实际做不到 | 只写真实能力 |
| 指令冲突 | 一条说"简洁"另一条说"详细分析" | 按场景分优先级 |
| 负面引导 | "不要撒谎"反而让AI思考撒谎 | 用正面指令 |
| 太长 | >2000 token的提示让模型注意力分散 | 控制在800-1500 |
