# 场景 Prompt 合集

日常开发中高频使用的 prompt 模板，拿来就用。

---

## 代码审查 prompt

```
你是一个严格的代码审查者。审查以下代码：

{代码}

请检查：
1. 逻辑正确性（边界条件、异常处理）
2. 安全性（注入、硬编码、权限）
3. 可维护性（命名、职责单一、重复代码）

按 P0(必须修) / P1(建议修) / P2(可选) 分级输出。
```

## 调试 prompt

```
我有一个 Bug：

现象：{现象}
代码：
```{语言}
{代码}
```
错误信息：{错误信息}

请：
1. 列出 3-5 种可能的根因，按概率排序
2. 对每种给出验证方式
3. 确定根因后给出修复方案
```

## 代码简化 prompt

```
简化以下代码。规则：
1. 不改变行为（输入相同→输出相同）
2. 删除重复/死代码
3. 理顺命名和结构
4. 保持功能完整

{代码}
```

## 代码转文档 prompt

```
为以下代码生成文档，包含：
1. 函数/类的作用（一句话）
2. 参数说明（类型、含义、默认值）
3. 返回值
4. 使用示例

{代码}
```

## 文章润色 prompt

```
润色以下文章，保持原意：

{文章}

要求：
1. 保持原文风格
2. 优化句式结构
3. 修正用词
4. 检查逻辑连贯性
```

## Bug 报告 prompt

```
分析以下报错：

错误信息：{错误信息}
相关代码：{代码}
最近改动：{刚改了什么}

请判断：
1. 这是新 bug 还是遗留问题？
2. 根因可能是什么？
3. 修复难度和影响范围？
```

## API 设计 review prompt

```
Review 以下 API 设计：

端点：{method} {path}
请求：{请求格式}
响应：{响应格式}

请检查：
1. RESTful 规范（命名/方法/状态码）
2. 安全性（认证/限流/参数校验）
3. 兼容性（向后兼容吗？）
4. 完整性（错误处理/分页/字段覆盖）
```

## 数据库迁移 prompt

```
分析以下数据库迁移方案：

当前表结构：{当前Schema}
目标表结构：{目标Schema}
存量数据量：{行数}

请评估：
1. 迁移对现有系统的影响
2. 数据一致性保障方案
3. 回滚策略
4. 推荐的迁移步骤
```
